AI Grading & Fraud Detection (Module 1). A customer initiates a return via structured, category-specific Q&A flow. Simultaneously, a local computer-vision pipeline grades the item from images/video, and for clothing & footwear, a Social Connect scanner checks public posts during the ownership window. The result is a Product Health Card: a machine and human-readable grade (Excellent / Good / Fair / Poor, score 0–100) with a recommended disposition (resell / refurbish / donate / recycle / return to seller) and a fraud signal. If wardrobing is detected (fraud confidence ≥ 0.60), the customer is offered P2P resale for Green Credits instead of a standard return, no accusation, just a better offer.

Grading pipeline (Module 1): Three components run concurrently via asyncio.gather. A local computer vision pipeline inspects submitted images for anomalies, defects, and wear patterns — the anomaly detector can run on a cutting-edge, training-free DINOv2 backend (ViT-S/14 with registers) that compares image patch features against a per-category "known-good" memory bank and ensembles pessimistically with the OpenCV heuristic (final severity = max(model, heuristic)), opt-in via ANOMALY_BACKEND=dinov2 with graceful fallback to the heuristic when torch/weights are unavailable — while a structured Q&A flow captures the customer's self-reported condition, both run in parallel and their results are combined to produce the final health score: 100 − (w₁·anomaly_severity + w₂·defect_penalty + w₃·return_reason_penalty + w₄·wear_detection_penalty). A cross-validation layer takes max(intent_penalty, wear_penalty) as the authoritative wear signal, ensuring that self-serving Q&A answers cannot override physical evidence from CV. Total pipeline budget: 2000ms. Once the score is computed, it is routed through the Condition Routing gate: items scoring above 90 are marked for resell, above 70 for refurbishment (only Electronics), above 50 for donation, and 50 or below for recycling.
